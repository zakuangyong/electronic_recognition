# 图纸比对页面重构（方案A：并排联动 + 差异叠加）

## 背景与问题

当前图纸比对页（[DrawingDiffView.vue](file:///d:/project/electronic_recognition/web/src/views/DrawingDiffView.vue)）以“上传 → 标注预览 → 差异清单”为主。虽然信息完整，但用户在核对时会遇到：

- 新旧图纸本体不可同时对照，视觉核对需要在脑内“切换”
- 差异清单与图纸定位之间的闭环弱（缺少双侧同步定位与可视化锚点）
- 标注页可看但不易做到“逐项确认”，难以形成工程核对节奏

本次改造目标：把“结果展示页”升级为“差异核对工作台”，强化新旧两版图纸的可视化差异标注与对比。

## 目标

- 提供“旧版/新版并排联动”的主对比视图（平移/缩放/页码联动）
- 将 diff_items 转化为“可定位的标注层”：bbox 高亮 + 编号锚点 + 选中态强调
- 右侧差异清单成为主导航：点击条目双侧同步定位并高亮，支持上一处/下一处
- 保留现有下载与汇总能力（JSON/Excel），但降低其在主流程中的占比

## 信息架构（IA）

- 顶部状态条
  - 任务状态、文件类型、页码、差异数量
  - 导航：上一处/下一处、复位视图、快捷键提示
- 左侧输入与参数（轻量）
  - 旧版/新版文件卡片
  - DPI/阈值/文件类型（高级参数可折叠）
- 中部主画布（核心）
  - 左：旧版图纸 viewport
  - 右：新版图纸 viewport
  - 同步：平移/缩放/页码
  - 叠加：bbox、编号、选中态高亮（弱化其他）
- 右侧差异清单（导航）
  - 搜索：ID/文本/页码/类型
  - 过滤：图形变化 / 文本变化 / 图文变化
  - 列表项：缩略图 + old/new 文本并排 + 元信息（页码、类型）

## 交互与状态

### 画布联动

- 默认开启同步：任意一侧操作平移/缩放，另一侧同步
- 支持开关同步，用于极端场景下独立查看
- 复位：回到默认缩放与位移
- 定位：选中差异项后，视图自动居中到 bbox 中心点

### 差异叠加层

- 使用 bbox 形成半透明高亮框，并附加编号锚点
- 支持开关叠加
- 支持“弱化其他”：只强调当前项，其余项降低透明度
- 颜色区分（建议）
  - visual_change：偏红
  - text_changed：偏青
  - graphic_or_text：偏绿

### 差异清单

- 点击条目：切换 active 差异项 + 双侧画布定位 + 高亮
- 上一处/下一处：顺序遍历 active
- 搜索与过滤影响“可见列表”与“叠加层显隐”

## 数据映射（复用现有接口）

现有类型定义见 [diff.ts](file:///d:/project/electronic_recognition/web/src/types/diff.ts)：

- `diff_items[]`：
  - `id` → region ID（用于清单键值与叠加层 data-id）
  - `page` → 页码
  - `bbox` → bbox 坐标（用于叠加层定位与视图 focus）
  - `crop_image_url` → 清单缩略图
  - `old_text` / `new_text` → 清单的 old/new 文本并排
  - `changed_type` → 过滤分类与颜色语义
- `annotated_images[]`：
  - 可作为“辅助预览”，但不取代并排联动主视图
- `downloads`：
  - 保留按钮入口（JSON 汇总 / Excel 报告），不作为主流程核心

## 成功标准

- 用户可以在一次视线移动内完成：清单 → 定位 → 新旧并排核对 → 下一项
- 不依赖 annotated_images 也能完成核对
- 关键视觉语义清晰：新/旧对照明确、差异项编号可追踪、当前项高亮明显
