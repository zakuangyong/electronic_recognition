# 工作台右侧面板替换设计

**日期**: 2026-06-25
**目标**: 将 `识别工作台` 右侧当前的 `结果摘要` 面板，替换为 `元件识别列表` 面板。

## 需求确认

用户已明确要求：

- 右侧显示 `元件识别列表`
- 每条只显示三段信息：
  - 元件代号
  - 元件名称
  - 数量
- 不显示：
  - 第 N 页
  - 任务状态摘要
  - 同步信息
  - 规则命中、预览页数等摘要字段

## 数据组织

右侧列表不直接逐条平铺原始 `detected_components`，而是先做聚合：

- 聚合键：`元件代号 + 元件名称`
- 聚合结果：
  - `code`: 元件代号
  - `label`: 元件名称
  - `count`: 该组合出现次数

这样右侧面板更适合作为辅区展示，不会因为同类元件重复出现而显得拥挤。

## UI 设计

### 面板标题

- 主标题：`元件识别列表`
- 标签：`components`

### 列表项结构

每项展示：

- 左侧：
  - 元件代号
  - 元件名称
- 右侧：
  - 数量

其中数量应是视觉重点，但不能超过当前方案 C 的右侧辅区权重。

### 空状态

当无识别结果时显示：

- `等待识别`
- `识别完成后，这里列出已识别元件`

## 实现范围

主要修改：

- `web/src/components/workbench/TaskStatusCard.vue`
  - 改造成聚合后的元件识别列表面板
- `web/src/views/WorkbenchView.vue`
  - 不再传递摘要字段，改为传递 `components`
- `web/src/views/__tests__/WorkbenchView.test.ts`
  - 更新断言，验证右侧已显示 `元件识别列表`

## 非目标

本次不做：

- 调整左侧主画布结构
- 调整 `ResultTabs` 的 tabs 逻辑
- 展示页码
- 展示置信度
- 新增交互筛选或排序

## 验收标准

视为完成，需要满足：

- 右侧面板标题已从 `结果摘要` 变为 `元件识别列表`
- 列表项只显示 `元件代号 / 元件名称 / 数量`
- 同类元件已按 `代号 + 名称` 聚合
- 不再出现页码和状态摘要字段
- 工作台测试继续通过
- `pnpm vue-tsc --noEmit` 继续通过
