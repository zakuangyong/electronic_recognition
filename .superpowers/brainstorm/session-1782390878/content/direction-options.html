中枢控制台

推荐方案。三页共享统一骨架：左侧导航，中间主作业区，右侧状态与辅助信息。利于把“操作”和“反馈”拆开。

优点

  • 结构统一，后续落 Vue 容易复用
  • 适合识别任务、知识编辑、检索结果三种重交互页面
  • 专业稳健，适合内部平台长期演进

注意点

  • 需要用层次和留白避免显得过于“后台”

流程编排台

把三页都设计成“阶段推进”模式，强调从上传到识别、从筛选到编辑、从查询到比对的流程感。

优点

  • 上手直观，适合强调操作闭环
  • 很适合做步骤引导与新手教学

注意点

  • 知识库和检索页会被“流程化”得略重
  • 对高频用户来说切换成本更高

树 / Facet / 标签

高密度分析屏

强调“专业分析工具”气质，信息量更高，更像检索与规则分析系统，而不是传统后台。

优点

  • 适合复杂检索、规则测试、结果比对
  • 专业感强，容易做出行业工具特色

注意点

  • 识别工作台第一页会显得较重
  • 需要更强的信息层次设计，否则容易压迫

建议

建议优先选 A 中枢控制台，再在图纸检索页局部吸收 C 的高密度分析能力。
